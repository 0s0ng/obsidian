# 네트워크란?

## 정의

- 두 대 이상의 컴퓨터 또는 장치가 데이터를 주고받을 수 있도록 유무선으로 연결된 통신망

## 목적

- 자원 공유 (리소스를 여러 장치가 공유하여 효율성 높임)
- 데이터 교환 (파일, 정보 교환)
- 정보 공유 (지식이나 기능을 공유하여 협업)

## 구성

- 노드(node: computer/network devices)와 링크(link: cable etc.)로 구성
	- 노드(Node)
		- 네트워크에 연결되어 데이터를 송수신 할 수 있는 모든 장치
	- 호스트(Host)
		- node의 타입 중 하나
		- 데이터 패킷의 출발지 또는 목적지가 되는 노드
	- 링크(Link)
		- 두 노드를 연결하는 물리적/논리적 통신 채널
		- 케이블, 무선 연결 등


# 네트워크 분류

## 범위별

- LAN(Local Area Network)
- ~~CAN(Campus Area Network)~~
- ~~MAN(Metropolitan Area Network)~~
- WAN(Wide Area Network)

이외에도 WLAN(Wireless LAN) 등 여러 종류가 존재하지만 일반적으로 LAN과 WAN이 있음

### LAN

- 근거리 네트워크(Local Area Network)
- ==가까운 거리에 위치한 비교적 소규모 네트워크==
- OSI layer 1, 2의 데이터 통신 장비(DCE)를 사용
- WAN에 비해 전파 지연이 적고 데이터 전송 속도 높음

### WAN

- 광역 네트워크(Wide Area Network)
- ==두 개 이상의 LAN을 넓은 지역에 걸쳐 연결하는 것==
- OSI layer 1,2,3의 DCE 사용
- 전파 지연이 길고 데이터 전송 속도가 느림

## 토폴로지별

### 네트워크 토폴로지(Network Topology)

- 네트워크에서 노드와 링크의 물리적/논리적 구성
- Ring, Mesh, Star, Tree, Bus 등의 형태가 존재

### 버스 토폴로지(Bus Topology)

- 네트워크상의 모든 장치가 ==하나의 케이블로 연결==됨
- 하나의 노드에서 데이터 전송 시 브로드캐스트 되어 다른 모든 노드가 수신할 수 있음
- 장점 
	- 간단한 설치 및 적은 비용
- 단점 
	- 케이블에 장애 발생 시 전체 네트워크 마비
	- 데이터를 양방향으로 동시에 보낼 수 없음

### 링 토폴로지 (Ring Topology, Token Ring)

- ==좌우의 인접한 노드와 연결==되어 있는 단 방향 전송 형태
- 한 번에 하나의 컴퓨터에 데이터를 전송하여 충돌이 발생하지 않음
- 장점 
	- 단순한 구조로 설치와 재구성 용이
- 단점 
	- 하나의 노드에 문제 발생 시 전체 네트워크 마비
	- 노드 재구성 또는 추가/삭제 시 절차가 복잡하고 전체 네트워크를 중단해야함

### 스타 토폴로지(Star Topology)

- 가장 일반적으로 사용되는 네트워크 토폴로지
- 네트워크 ==중앙에 위치한 노드가 모든 노드를 연결==
- 장점 
	- 하나의 노드에 장애가 발생하더라도 나머지 네트워크는 영향받지 않음
	- 중앙 노드만이 다른 노드와의 통신을 제어하므로 데이터 충돌 방지 가능
- 단점
	- 중앙 노드에 문제 발생 시 전체 네트워크 마비
	- 최초 네트워크 구성 시 소요 비용이 큼

### 트리 토폴로지(Tree Topology)

- 트리 형태의 노드에 장치를 두어 노드들을 연결하는 형태(==부모-자식 계층 구조로 연결==, 계층적인 네트워크)
- 장점 
	- 관리와 확장에 용이
	- 여러 컴퓨터를 분리하거나 우선순위 부여 가능
- 단점 
	- 중앙에 트래픽이 집중되어 병목현상 발생 가능
	- 중앙 노드에 문제 발생 시 전체 네트워크 마비

### 메쉬 토폴로지(Mesh Topology)

- 중앙에서 제어하는 장치 없이 ==모든 노드가 점 대 점으로 상호 연결==된 구조
- n(n-1)/2 개의 물리적 채널 필요
- 장점 
	- 보안에 유리
	- 특정 노드에 장애가 발생하더라도 전체 시스템에 영향을 주지 않음
- 단점 
	- 복잡한 구성
	- 많은 수의 케이블 필요
	- 노드 추가 시 많은 비용부담 발생

## 역할별

### Client-Server

- 컴퓨터와 컴퓨터가 통신하는 구조(N:1)
- ==클라이언트가 요청하면 서버가 응답==하는 방식
- 대부분의 웹 사이트에서 사용되는 방식
- 중앙화
- 클라이언트
	- 서비스를 요청하는 시스템
	- 데스크톱, 노트북, 스마트폰, 태블릿 등
- 서버
	- 서비스를 제공하는 시스템
	- db 서버, web 서버, file 서버, dns 등
- 장점
	- 중앙화된 컴퓨팅 방식으로 서버가 모든 접근과 데이터를 관리하여 보안상 우수하며 안전함
	- 플랫폼에 구애받지 않아 클라이언트와 서버가 다른 운영체제를 사용하여도 괜찮음
- 단점
	- 한 번에 너무 많은 클라이언트가 요청을 하면 서버가 뻗을 수 있음

### P2P(Peer-to-Peer)

- ==모든 컴퓨터가 서로 데이터를 주고받을 수 있는 네트워크 구조(n:n)==
- 각 컴퓨터를 노드(node) 또는 피어(peer)라고 부름
- 예 : 온디스크, 토렌트, 비트코인 거래
- 탈중앙화(분산화)
- 장점
	- 네트워크 구축을 위한 많은 하드웨어가 필요하지 않음
- 단점
	- 검증되지 않은 사용자와의 통신 과정에서의 안전을 담보하기 어려움
	- 조직적으로 파일을 유지/관리하기 어려움


# 네트워크 통신 방식

|   타입   | 통신 대상 |   범위    |     예제     |
| :----: | :---: | :-----: | :--------: |
| 유니캐스트  |  1:1  | 전체 네트워크 | HTTP, SSH  |
| 멀티캐스트  | 1:그룹  | 정의된 구간  | IPTV, 화상회의 |
| 브로드캐스트 | 1:전체  |   LAN   | ARP, DHCP  |
| 애니캐스트  |  1:1  | 전체 네트워크 | DNS, IPv6  |

- 유니캐스트(Unicast)
	- 1:1 통신 방식 (가장 일반적인 통신 방식)
	- 서버와 클라이언트 간의 통신에 주로 사용
- 브로드캐스트(Broadcast)
	- 1:전체 통신 방식 (같은 LAN 내 모든 장치)
- 멀티캐스트(Multicast)
	- 1:그룹 통신 방식 (멀티캐스트 구독 호스트)
- 애니캐스트(Anycast)
	- 1:1 통신 방식(동일 그룹 내의 1개 호스트)
	- 다수의 동일 그룹 중 가장 가까운 호스트에서 응답
